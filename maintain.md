# 维护说明

这个仓库的维护目标是清楚、可回看、能订阅。宁可慢一点人工确认，也不把自动采集出来的结果直接塞进主订阅。

## 文件边界

| 文件或目录 | 用途 | 维护方式 |
| --- | --- | --- |
| `public/index.m3u` | 唯一主订阅文件 | 人工编辑、人工复核 |
| `public/logos/` | 频道台标 | 文件名尽量与频道名一致 |
| `reports/` | 本地检测报告和人工记录 | 由 Git 忽略，只在本地保留，不对外发布 |
| Markdown 页面 | 使用和维护说明 | 让下一个维护者能看懂 |

公开文档只记录复核结论和口径，不直接链接 `reports/`。需要交接原始报告时，应通过单独的归档渠道传递，避免把大量截图、扫描日志和中间 CSV 混入主仓库。

## M3U 字段

每个频道条目尽量保持下面几个字段稳定：

| 字段 | 说明 |
| --- | --- |
| `tvg-id` | 与频道名保持一致，方便播放器识别 |
| `tvg-name` | 频道显示名，需和逗号后的显示名一致 |
| `tvg-logo` | 指向 Pages 上发布的台标 |
| `group-title` | 基础分组，例如央视频道、河南频道、河南地市 |

## 分组原则

- 央视、CGTN、CETV 等放入 `央视频道`。
- 省级卫视放入 `地方卫视`，并按照地方区号排序。
- 河南省级频道放入 `河南频道`。
- 郑州、开封、洛阳等地市频道放入 `河南地市`，并按照地方区号排序。
- 付费、专题、少儿、纪实、影视等放入 `数字频道`。

## 提交前检查

1. `tvg-name` 与显示名一致。
2. 同一个 URL 不跨频道重复。
3. 台标文件存在，路径能被 Pages 发布。
4. `CCTV16` 等用户明确要求保留的频道不要因为抽帧失败而删除。
5. 运行 `npm run check`，确认文档站可以构建。

## 发布说明

当前 GitHub Pages 只在 `main` 分支变更后自动发布。普通 `codex/` 分支推送只用于查看代码，不会让公开站点立即变化。
