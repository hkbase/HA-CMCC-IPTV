# 播放排查

先把问题分成两层：订阅文件能不能下载，直播源能不能拉流。前者走 GitHub Pages，后者走河南移动 IPTV 网络。

## 快速判断

| 检查项 | 正常表现 | 异常时说明 |
| --- | --- | --- |
| 打开订阅地址 | 浏览器下载或显示 M3U 文本 | 地址错误、Pages 未发布或网络访问 GitHub 失败 |
| 播放器导入订阅 | 出现频道列表 | 播放器不支持远程 M3U，或地址未复制完整 |
| 单独打开频道 URL | VLC/PotPlayer 开始缓冲播放 | 当前网络无法访问 IPTV 源，或该源已失效 |
| 台标显示 | 频道旁出现图标 | 播放器不支持、缓存未刷新或图片路径被拦截 |

## 常见症状

### 频道列表为空

1. 确认地址是完整的 `https://elykia093.github.io/HA-CMCC-IPTV/index.m3u`。
2. 用浏览器打开订阅地址，看是否能得到 `#EXTM3U` 开头的文本。
3. 换 VLC 或 PotPlayer 导入，排除播放器兼容问题。

### 全部频道都不能播放

这通常不是 M3U 格式问题，而是网络边界问题。当前源主要依赖河南移动 IPTV 网络，普通公网或其他运营商无法访问很常见。

优先检查：

- 当前出口是否为河南移动宽带。
- 是否启用了代理、加速器、旁路由或会改出口的 DNS。
- 路由器是否把播放器流量转到了其他网络。

### 部分频道不能播放

单条源可能随运营商侧调整失效。先到 [频道列表](/channels) 看这个频道是否还有其他源数；如果有多条源，播放器通常会按列表顺序尝试。

### 实际播放和频道名不一致

这种属于错台。维护时会优先处理明确错台源：能确认实际播成其他频道的，移动到实际频道；实际播成无关频道且不适合保留的，从原频道删除。

## 不建议的判断方式

- 只看网页能不能打开。网页能打开不代表 IPTV 源能访问。
- 只看台标是否显示。台标和播放流是两套资源。
- 在非河南移动网络下判断全量失效。这个环境本身就可能无法访问。
